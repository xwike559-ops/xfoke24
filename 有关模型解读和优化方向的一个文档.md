### 存在的问题

串行结构容易出现：CNN 提的细节在后续注意力里被“平均化”或被错误区域抢走权重

融合决策（mask）容易不稳定，清晰区域边界容易糊/抖

#### HybridFusionNet



- **为什么 patch_size=2？**
  - 多聚焦融合很怕丢细节，patch 太大（如 4/8）会模糊边界
  - 2 是一个“细节保留”和“计算量可控”的折中

虽然只堆了 2 层，但已经能实现“跨窗口信息交换”；如果堆 4/6/8 层，全局性会更强。

#### Lossfunction

**① 梯度最大值保留（核心“清晰度”约束）**

```python
g1 = sobel(img1)
g2 = sobel(img2)
gf = sobel(fused)
grad_max = max(g1, g2)
loss_grad = L1(gf, grad_max)

```

融合图的边缘强度 gf
 应该尽量接近两张输入图中更清晰的那一张（梯度更大的一张）

Sobel 是“粗糙梯度”，对噪声也敏感
 显微图像（类器官）里噪声多，可能把噪声当作“清晰细节”

**② 像素强度一致性（保证亮度/能量）**

```
intensity_target = max(img1, img2)
```

**潜在问题（尤其显微）**：

- 显微成像里“亮”不等于“清晰”
- 细胞/类器官很多结构是暗纹理，max 强度可能把背景抬高、造成偏亮

**③ mask 正则：防止全 0 / 全 1（鼓励“有过渡”）**

##### 损失函数改进：

Lytro 推荐（简单强力版）

1. `loss_detail = L1( LoG(fused), max(LoG(img1), LoG(img2)) )`
2. `loss_ssim = 1 - SSIM(fused, target_soft)`（target_soft 可用清晰度加权）
3. `loss_mask_bin = mean(mask*(1-mask))`
4. `loss_mask_tv = TV(mask)`（可选）

------

✅ 类器官/细胞显微推荐（更稳更贴合）

1. **清晰度加权目标**（Tenengrad/Laplacian energy 生成 w）
2. `loss_detail = Charbonnier( Lap(fused) - Lap(target_soft) )`
3. `loss_lowfreq = L1(blur(fused), blur(0.5*img1+0.5*img2))`（背景一致）
4. `loss_mask_tv = TV(mask)`（强烈推荐）
5. `loss_mask_bin = mean(mask*(1-mask))`（权重调小，防止过早二值化）

## v2模型解读

![image-20260121201007788](C:\Users\14875\AppData\Roaming\Typora\typora-user-images\image-20260121201007788.png)

- 每个 stage：CNN 分支稳定产出局部结构
- 同时把两路特征 `cat(f1,f2)` 丢进 Swin token mixer，让注意力在**对比信息**里学“哪里更重要”
- 然后用引导特征 `guide` 回注两路分支，再进入下一层

**为什么这比串行好**：

- 串行：Transformer 只能看到一条“已经被卷积加工过的结果”
- 并行：Transformer 直接看到 `f1 vs f2` 的对比，天然更适合做“选择/融合”这种任务

> 并行不是指“Transformer直接吃原图”
>
> ### 正确的“并行”含义是：**两条特征分支同时存在并被同时优化**
>
> - **CNN 分支**：每张图各自走 CNN（保持各自细节表征）
> - **Transformer 分支（token mixer）**：在同一尺度上，对 `cat(f1,f2)` 做跨区域对比与交互
>
> 注意：这里的并行是**功能并行/分支并行**，不是“输入并行”。
>
> **得到的 tr 表示什么：**
>
> > Transformer 在“对比输入 cat”上做了跨区域仲裁后产生的“融合引导特征”，强调该关注的区域与差异。

**收益**：

- 多尺度一致性更强（512→256→128→64）
- mask 学得更像“选择图”而不是“平滑混合”

> Decoder 是“翻译器”：把高维特征翻译成“每个像素选谁”。

随着stage的递增，分辨率递减、感受野更大，语义从“边缘纹理”逐渐过渡到“区域结构”

```
guide = (f1 + f2 + tr)/3

f1：来自图1的局部细节候选

f2：来自图2的局部细节候选

tr：来自 Transformer 的全局对比/仲裁意见

取平均的意义是：

不让某一路支配（稳定）

把“局部细节 + 全局决策”融合成一个对后续有用的指导信号
```

#### **解码--mask**

```python
mask_logits = self.mask_head(x)      # (B,1,512,512)
mask = torch.sigmoid(mask_logits)
fused_image = mask * img1 + (1 - mask) * img2

//mask_head:Conv3x3(C→C) → ReLU → Conv1x1(C→1) → sigmoid
//3×3：负责“空间上的形状与连续性”（让决策在边界处更合理、更平滑）
//1×1：负责“像素级最终判决”（把每个像素的 C 维证据压成一个 logit）
```

 **1×1 卷积写成“数学形式”**:

> mask 本质是一个“软决策图（soft decision map）”



## v3模型解读

![image-20260224103211976](C:\Users\14875\AppData\Roaming\Typora\typora-user-images\image-20260224103211976.png)

**引入 DeepLabv3 语义先验（从“纯低层视觉”到“语义引导差异聚焦”）**

**用语义分割产生的权重，引导特征集中在“近焦与远焦差异显著”的区域**

M 重新定义为：

- `D`：近/远焦清晰差异图（Laplacian/Gradient 差异）
- `S`：语义结构先验（DeepLabv3 输出）
- **最终**：`M = Normalize( D ⊙ (1 + λ·∇S) )^γ`

> 直觉解释：
>
> - **差异大**（远近焦之间变化明显、边缘/纹理明显）的区域，M 应该高；
> - **语义强**（物体区域、结构边界）的位置，M 也更值得高。
>    最终你把这些融合成一张归一化到 [0,1] 的 M
>
> 不是在 feature 上“乘个权重图”，而是在注意力的**竞争机制**里提前“给高 M 的 token 加分”，让它们在全局信息交互时天然占优势。

并把它接入 Swin 注意力：

- softmax 前 logits bias（最稳定的注入方式）
- 且让 M 跟随 shift/window_partition，保证 SW-MSA 对齐



#### 改进点：

##### 1) 先验注入方式：现在是 **logits 偏置**，但偏置的形状太“温柔”

你目前的语义偏置是：

- `bias_ij = 0.5*(m_i + m_j)`，然后 `attn += prior_beta * bias`，发生在 softmax 之前（非常对） 
- `prior_beta` 可以设置成可学习参数（也很对）

**问题：** `0.5*(m_i+m_j)` 只是在所有 pair 上加一个平滑的“整体增益”，它更像“提高某些 token 的人气”，但并不强迫注意力去“看差异区域”。
 **建议两种升级（选其一即可）：**

1. **用 logit 形态增强（更尖锐）**：你代码里已经留了注释 `bias = log(bias+eps)`，建议真的打开并配合一个温度系数，让高 M 的区域在 softmax 前更“拉开差距”。 swin2d_parts
2. **改为 pairwise 差异偏置**：让 `bias_ij` 与 |m_i - m_j| 相关（抑制跨“不同显著性”的连接），这样会更符合“聚焦/散焦边界”决策的物理直觉。

##### 2)先验 M 的构造：你只用 img1 做语义，容易引入“偏置”

你在先验里写了：

- `S_conf = self._semantic_conf(img1)`（只跑 img1）
- 语义模型是冻结的 DeepLabv3，输出 top-1 概率置信度并插值回原图

**问题：** 对多聚焦场景，img1/img2 的清晰区域不同，**只用 img1 做语义**会让 M 在某些区域天然偏向 img1（尤其是你训练数据是合成模糊时，模糊会影响分割置信度）。
 **建议：** 改成 `S_conf = semantic_conf((img1+img2)/2)` 或 `max(conf1, conf2)`，让语义先验更“中立”。

##### 3) Cross-Attention bottleneck：你现在的 t2 用 `flip` 是个“占位技巧”，该换成真正的双域 token

你 bottleneck 做 cross-attn 的部分是：

- `t = b.flatten(2).transpose(1,2)`
- `t1=t`，`t2=torch.flip(t, dims=[1])` 然后 cross-attn hybrid_fusion_net

这会带来一个隐患：**cross-attn 其实没有真正建立 img1 token ↔ img2 token 的对应关系**，更像在做一个“自扰动注意力”。

**更好的做法（很简单）：**

- bottleneck 前你有 `s4_f1`、`s4_f2`，直接把它们各自投影成 token：
   `t1 = proj(s4_f1).flatten(2).T`，`t2 = proj(s4_f2).flatten(2).T`
- 然后做 cross-attn（这才是你想要的“远/近焦对话”）

##### 4) 训练稳定性：prior_beta 建议“分层/分阶段”

你现在的 prior 注入是 **所有 Swin stage 都用同一个 prior_beta**（通过 PatchTokenMixer 传进去）。 hybrid_fusion_net

经验上更稳的做法是：

- **浅层（高分辨率）prior 强一点**：帮助边界和细粒度区域更快对齐
- **深层（低分辨率）prior 弱一点**：避免语义先验过度支配全局结构

实现方式：让 tr1~tr4 各有自己的 `prior_beta`（或 learnable_prior=True 但加一个 stage-wise 初值）。

```markdown
│ 12 │ 13 │ 14 │ 15 │

窗口大小: 2×2
```

2. 常规窗口划分 (W-MSA) - 无移位

```markdown
常规窗口 (shift_size=0):
┌───┬───┐┌───┬───┐
│ 0 │ 1 ││ 2 │ 3 │
│ 4 │ 5 ││ 6 │ 7 │
└───┴───┘└───┴───┘

```

**注意力计算**：每个窗口内部计算自注意力，窗口间**不通信**

3. 移位窗口划分 (SW-MSA) - 关键步骤

```markdown
移位 (shift_size=1, 向右下移动1个像素):

移位后特征图：
┌────┬────┬────┬────┐

├────┼────┼────┼────┤
│ 1  │ 2  │ 3  │ 0  │
└────┴────┴────┴────┘
```

4. 在移位特征图上划分窗口 (2×2)

```markdown
移位后的窗口划分：
┌───┬───┐┌───┬───┐
│ 5 │ 6 ││ 7 │ 4 │
│ 9 │ 10││ 11│ 8 │
└───┴───┘└───┴───┘
┌───┬───┐┌───┬───┐
│ 13│ 14││ 15│ 12│
│ 1 │ 2 ││ 3 │ 0 │
└───┴───┘└───┴───┘

窗口1: {5,6,9,10}   窗口2: {7,4,11,8}
窗口3: {13,14,1,2}  窗口4: {15,12,3,0}
```

5. 问题来了：不自然的窗口组合

**窗口3包含不相邻的像素**：

- 位置13, 14（原特征图的第3行）
- 位置1, 2（原特征图的第0行）

在**原始空间**中，位置1和13不相邻，不应该互相计算注意力！这就是需要**掩码**的原因。

掩码**不是**阻止窗口间信息交换，而是**精确控制**交换的方式。它确保**只有相邻的原始窗口之间能交换信息**，而不是让所有像素随意混合。

- #### `patch_size` 的参数意义是什么？

  - **每个 token 对应原特征图上多大的一块区域（patch）**
     同时也是 **下采样倍率**（stride=patch_size）

- **将注意力头数量（num_heads）增加会有什么影响？**
  - **影响 1：表达能力变化（更“细分视角”）**
    - 多头注意力可以理解为：**同时用多种“相似度度量方式”去看同一个窗口**
    - heads 越多，模型更容易学到不同类型关系（边缘、纹理、结构、对比度等）
  - **每个 head 的维度会变小（重要）**
    - ead_dim 太小可能导致每个头的信息容量不足，反而效果不一定更好。
- **为什么 Transformer 部分对初始化很敏感？**
  - Transformer 里面有三类“敏感算子”，一旦初始权重尺度不合适，就容易训练不稳（梯度爆炸/消失、softmax 饱和等）。

尺寸适配（关键）：
Swin Transformer 对图像尺寸有严格要求（必须是 window_size 的倍数）。你的原始图像是 520x520。
如果 PatchSize=2, WindowSize=8，特征图大小为 260，260/8 = 32.5（不整除，会报错）。
解决方案：我在 MultiFocusDataset 中将 resize 默认设为 512x512（CV领域标准尺寸，且被8整除）。这是最稳健的做法，避免复杂的动态Padding导致的边缘伪影。
依赖项：
代码依赖 timm 库（PyTorch Image Models），这是Swin Transformer的标准实现库。如果未安装，请先运行 pip install timm。

不仅显示融合图，现在还会显示决策图（Decision Mask）。这在写论文时是证明模型“真的学到了聚焦区域”的最强证据。
