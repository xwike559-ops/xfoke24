**引入 DeepLabv3 语义先验（从“纯低层视觉”到“语义引导差异聚焦”）**

**用语义分割产生的权重，引导特征集中在“近焦与远焦差异显著”的区域**

M 重新定义为：

- `D`：近/远焦清晰差异图（Laplacian/Gradient 差异）
- `S`：语义结构先验（DeepLabv3 输出）
- **最终**：`M = Normalize( D ⊙ (1 + λ·∇S) )^γ`

> 直觉解释：
>
> - **差异大**（远近焦之间变化明显、边缘/纹理明显）的区域，M 应该高；
> - **语义强**（物体区域、结构边界）的位置，M 也更值得高。
>    最终你把这些融合成一张归一化到 [0,1] 的 M
>
> 不是在 feature 上“乘个权重图”，而是在注意力的**竞争机制**里提前“给高 M 的 token 加分”，让它们在全局信息交互时天然占优势。

并把它接入 Swin 注意力：

- softmax 前 logits bias（最稳定的注入方式）
- 且让 M 跟随 shift/window_partition，保证 SW-MSA 对齐





#### 当前架构

> # 当前模型架构描述
>
> ## 模型概述
>
> 本模型名为 HybridFusionNet，旨在实现多聚焦图像融合任务，通过结合卷积神经网络（CNN）、Swin Transformer 和语义先验引导机制，实现对两张输入图像（近焦和远焦）的像素级融合。该模型采用端到端无监督学习框架，输出融合图像和决策掩码（mask），其中掩码指示每个像素应从哪张输入图像选择。模型架构分为语义先验模块、编码器（CNN + Swin Transformer）、瓶颈层（伪交叉注意力）和解码器四个主要部分。以下详细描述各模块及其配合关系。
>
> ## 1. 语义先验模块 (SemanticPriorM)
>
> 语义先验模块是模型的核心创新之一，用于生成单通道先验图 M（Prior Map），该图在聚焦差异显著区域（如边界和细节）具有较高值，从而引导后续注意力机制聚焦于关键区域。该模块不参与最终融合，而是作为注意力偏置注入 Transformer。
>
> ### 1.1 输入处理
>
> - 输入：两张 RGB 图像 img1 和 img2（形状为 (B, 3, H, W)，值域 [0, 1]）。
> - 首先，将 img1 和 img2 转换为灰度图 g1 和 g2，使用标准 RGB 到灰度转换公式：g = 0.2989 * R + 0.5870 * G + 0.1140 * B。
>
> ### 1.2 聚焦差异计算 (Focus Difference)
>
> - 根据配置参数 diff_mode 计算聚焦差异图 D：
>   - 若 diff_mode = "lap"，使用 Laplacian 算子计算灰度图的二阶导数幅度：D = |Laplacian(g1) - Laplacian(g2)|。
>   - 若 diff_mode = "grad"，使用 Sobel 算子计算梯度幅度：D = |∇g1 - ∇g2|。
> - 对 D 进行归一化，使其值域为 [0, 1]：D = (D - min(D)) / (max(D) + ε)，其中 ε = 1e-8 以避免除零。
>
> ### 1.3 语义置信度计算 (Semantic Confidence)
>
> - 使用预训练的 DeepLabv3-ResNet50 模型（若 use_deeplab = True）计算语义置信度：
>   - 输入图像为 (img1 + img2) / 2（平均图），以减少单图像偏置。
>   - 对输入进行标准化：x = (x - mean) / std，其中 mean 和 std 为 ImageNet 预训练统计值。
>   - DeepLabv3 输出 logits，经 softmax 得到概率图 prob (B, C, H, W)。
>   - 提取 top-1 概率作为语义置信度 S_conf = max(prob, dim=1)，形状为 (B, 1, H, W)。
> - 若 use_deeplab = False，则 S_conf 为全零张量。
>
> ### 1.4 语义引导融合 (Semantic Guidance)
>
> - 根据 semantic_mode 参数融合差异和语义信息：
>   - 若 semantic_mode = "edge"，计算语义边界强度 S_edge = ∇S_conf（使用 Sobel 梯度），然后 M = D * (1 + lam * S_edge)，其中 lam 为边界强化强度（默认 2.0）。
>   - 若 semantic_mode = "gate"，直接门控：M = D * S_conf。
> - 对 M 再次归一化至 [0, 1]。
> - 可选幂次增强：若 gamma ≠ 1.0，则 M = M^gamma（默认 gamma = 1.5），以聚焦差异区域。
>
> ### 1.5 输出
>
> - 输出 M：形状为 (B, 1, H, W)，值域 [0, 1]，用于后续注意力注入。
>
> 该模块在模型前向传播开始时一次性计算 M_full，并在各编码阶段插值至对应分辨率，作为注意力偏置。
>
> ## 2. 编码器部分 (Encoder: CNN Stages + Swin Transformer Mixers)
>
> 编码器采用混合架构，结合 CNN 的局部特征提取和 Swin Transformer 的全局建模能力，共四个阶段（stage1 到 stage4），分辨率依次减半（512 → 256 → 128 → 64）。每个阶段先通过 CNN 提取特征，再通过 Swin Transformer 进行跨图像交互，同时注入语义先验 M。
>
> ### 2.1 Stem 层
>
> - 输入 img1 和 img2 经共享的 stem 层：Conv2d(3, 64, 3, padding=1) + BatchNorm2d + ReLU，输出特征图 f1 和 f2（形状 (B, 64, H, W)）。
>
> ### 2.2 各阶段处理 (_stage 方法)
>
> - 对于每个阶段 (stage1 到 stage4)：
>   - **CNN 子模块**：使用 DenseConvBlock（两层 DenseConvBlock，每层包含三层卷积 + BN + ReLU），输入通道为 64，输出通道保持 64。处理 f1 和 f2，得到 s*_f1 和 s*_f2。
>   - **下采样**：除 stage1 外，使用 Conv2d(64, 64, 3, stride=2, padding=1) + BN + ReLU 将分辨率减半。
>   - **特征拼接**：cat = torch.cat([s*_f1, s*_f2], dim=1)，得到 (B, 128, H', W')。
>   - **Swin Transformer Mixer**：使用 PatchTokenMixer 将 cat 转换为 token 序列，应用 Swin Transformer 块进行自注意力，同时注入插值后的 M_stage。
>   - **投影和上采样**：通过 1x1 Conv 将通道从 128 降至 64，然后上采样回原分辨率（使用双线性插值）。
>   - **引导融合**：计算 guide = (s*_f1 + s*_f2 + tr) / 3.0，经 SpatialEnhanceModule（SE 模块：全局平均池化 + 1x1 Conv + Sigmoid）增强，然后 f1 = f1 + guide，f2 = f2 + guide，实现跨图像特征融合。
>
> ### 2.3 Swin Transformer Mixer (PatchTokenMixer)
>
> - **Patch Embedding**：将 (B, 128, H', W') 通过 Conv2d(128, embed_dim, patch_size, stride=patch_size) 转换为 token (B, L, embed_dim)，其中 L = (H'/patch_size) * (W'/patch_size)。
> - **Swin Blocks**：应用两个 SwinTransformerBlock2D（一个窗口注意力 + 一个移位窗口注意力），每个块包含 LayerNorm + WindowAttention + MLP + DropPath。
> - **注意力注入**：在 WindowAttention 中，将 M_stage 插值至 token 网格大小 (H_tok, W_tok)，转换为 (B, H_tok, W_tok, 1)，然后在 softmax 前添加偏置：attn += prior_beta * bias，其中 bias = log(0.5 * (m_i + m_j) + ε)，prior_beta 为可学习或固定参数。
> - **输出**：将 token 转换回特征图 (B, 128, H', W')，经 1x1 Conv 投影至 64 通道。
>
> 该编码器通过 CNN 提取局部细节，Swin Transformer 建模全局关系，语义先验 M 引导注意力聚焦差异区域，实现多尺度特征融合。
>
> ## 3. 瓶颈层 (Bottleneck: Pseudo Cross-Attention)
>
> 瓶颈层位于编码器末端（stage4 输出），用于进一步建模跨图像交互，但当前实现为伪交叉注意力（pseudo cross-attention），而非严格的双图像交叉注意力。
>
> ### 3.1 输入
>
> - 输入 cat4 = torch.cat([s4_f1, s4_f2], dim=1)，形状 (B, 128, 64, 64)。
>
> ### 3.2 Token 化
>
> - 通过 Conv2d(128, 192, 1) 提升通道至 192。
> - 展平为 token：(B, 192, 64, 64) → (B, L, 192)，L = 4096。
>
> ### 3.3 交叉注意力 (CrossAttentionFuse)
>
> - 使用 CrossAttentionFuse 模块：包含 MultiheadAttention (num_heads=6) + FFN (LayerNorm + Linear + GELU + Linear)。
> - 输入 t1 = t，t2 = torch.flip(t, dims=[1])（token 顺序反转，作为伪第二序列）。
> - 计算 a12 = MHA(query=t1, key=t2, value=t2)，a21 = MHA(query=t2, key=t1, value=t1)，输出 z = a12 + a21。
> - 经 FFN 和残差连接：z = LN(z + FFN(z))。
>
> ### 3.4 输出
>
> - 将 z 转换回特征图 (B, 192, 64, 64)，经 Conv2d(192, 64, 1) 输出至解码器。
>
> 该瓶颈模拟跨图像交互，但由于 t2 为 t1 的反转，而非独立从 img2 生成，实际为自注意力变体。未来可升级为真实交叉注意力。
>
> ## 4. 解码器部分 (Decoder)
>
> 解码器采用 U-Net 风格的上采样架构，从瓶颈特征逐步恢复分辨率和通道，最终生成掩码。
>
> ### 4.1 上采样阶段
>
> - 从 x = bottleneck_output 开始：
>   - **解码块**：使用 ConvTranspose2d(64, 64, 2, stride=2) 上采样至 128，然后 cat([x, s3_g])，经 Conv2d(128, 64, 3, padding=1) + ReLU。
>   - 重复上采样至 256 和 512，依次拼接 s2_g 和 s1_g。
> - 每个解码块输出 64 通道特征。
>
> ### 4.2 掩码头 (Mask Head)
>
> - 最终特征经 Conv2d(64, 64, 3, padding=1) + ReLU + Conv2d(64, 1, 1)，输出 logits。
> - 掩码 mask = sigmoid(logits)，形状 (B, 1, H, W)，值域 [0, 1]。
>
> ### 4.3 融合输出
>
> - 融合图像 fused = mask * img1 + (1 - mask) * img2。
>
> ## 5. 模块配合与整体流程
>
> - **语义先验与编码器**：M 在全分辨率计算，然后在各 stage 插值注入 Swin 注意力，引导特征提取聚焦差异。
> - **CNN 与 Transformer**：CNN 提供局部特征，Transformer 通过窗口注意力建模全局关系，SE 模块增强跨图像一致性。
> - **编码器与瓶颈**：编码器输出多尺度特征，瓶颈进一步交互，输出送至解码器。
> - **解码器与输出**：解码器从低分辨率重建掩码，掩码驱动像素级融合。
> - **训练**：使用 ImprovedFusionLoss（细节损失 + SSIM + 掩码二值化 + 掩码 TV 正则），优化器为 AdamW，学习率 1e-4。
>
> 该架构通过语义先验增强注意力，多尺度特征融合确保细节保留，伪交叉注意力模拟交互，解码器生成可靠掩码，实现高质量多聚焦融合。参数量约 10.5M，适用于 512x512 输入

#### 细节解析

##### 先验模型

> **五种低级结构先验**
>
> - LoG 差异先验。
>   - 二阶导数滤波器，检测像素局部强度变化的曲率
>   - 响应特征
>     - 在边界和纹理区域产生强响应
>     - 对高频成分敏感（清晰区域高频丰富，模糊区域低频）
>     - 天然强调细节和纹理
>   - 稳定性：对光照缓慢变化鲁棒
>   - 使用场景;
>     - **自然图像多焦距融合**：效果最好，特别是有丰富纹理的场景
>     - **文本/细节丰富区域**：边界定位精准
> - Gradient 差异先验。
>   - 一阶导数（Sobel 算子），计算相邻像素强度梯度
>   - 响应特征
>     - 边界处响应强，平坦区域响应弱
>     - 对一阶变化敏感（相比二阶更"温和"）
>     - 计算量低，速度快
> - Local contrast 先验。
>   - 计算固定窗口内的局部方差（标准差），反映该区域对比度
>   - 响应特征
>     - 对比度高的区域（清晰）更亮
>     - 平坦、低对比区域（模糊）更暗
>     - 窗口大小（kernel_size=7）决定"感受野"
> - 多尺度 focus measure 先验。
>   - 在多个尺度下分别计算"局部对比度 + 梯度"，然后平均
>   - 多尺度策略
>     - kernel_size=3：捕捉细小细节
>     - kernel_size=5：中等纹理
>     - kernel_size=9：大尺度结构
>   - **鲁棒性**：多尺度平均提高稳定性，减少单尺度噪声影响
>   - **特性**：结合对比度和梯度的互补信息
>
> **组合方式**
>
> - **语义边缘增强结构先验**
>   - `M = Normalize(D * (1 + lambda * S_edge)) ^ gamma`；语义不直接决定选图，只增强结构差异区域，比较稳。缺点：如果 D 本身很弱，语义边缘再强也难以补起来。
> - **门控式组合**
>   - `M = Normalize(D * S_conf)`；语义置信度低的区域会被压制，能减少无意义纹理噪声。缺点：可能误伤真实聚焦细节，尤其是 DeepLabv3 不认识的细胞/细碎结构。
> - **加权融合式组合**
>   - M = Normalize(alpha * D + (1 - alpha) * S_edge)
> - **自适应增强式组合**
>   - `M = Normalize(D * (1 + lambda * S_edge) + mu * D * S_conf)`
> - ps：s_conf 是全局的置信度，用于判断前景后景的可能性概率；s_edge是表示边缘强度的变化速率

> ##### 先验偏置模式
>
> - sum bias。
>   - 高值（0.8-1.0）间的注意力权重差异大
>   - 低值（0.0-0.2）被有效压制
>   -  **自然图像**（Lytro）：差异区域清晰，需要强烈集中注意力
>   - ✅ **标准多焦距融合**：推荐首选
>   - ❌ **低对比度图像**：可能过度抑制有用信息
>
> - product bias。
>   - bias = m_i * m_j
>   - 只有**都在差异区**时才有高值
> - log bias。
> - difference bias。
> - hybrid bias。

#### 改进点：

##### 1) 先验注入方式：现在是 **logits 偏置**，但偏置的形状太“温柔”

> 你目前的语义偏置是：
>
> - `bias_ij = 0.5*(m_i + m_j)`，然后 `attn += prior_beta * bias`，发生在 softmax 之前（非常对） 
> - `prior_beta` 可以设置成可学习参数（也很对）
>
> **问题：** `0.5*(m_i+m_j)` 只是在所有 pair 上加一个平滑的“整体增益”，它更像“提高某些 token 的人气”，但并不强迫注意力去“看差异区域”。
>  **建议两种升级（选其一即可）：**
>
> 1. **用 logit 形态增强（更尖锐）**：你代码里已经留了注释 `bias = log(bias+eps)`，建议真的打开并配合一个温度系数，让高 M 的区域在 softmax 前更“拉开差距”。 swin2d_parts
> 2. **改为 pairwise 差异偏置**：让 `bias_ij` 与 |m_i - m_j| 相关（抑制跨“不同显著性”的连接），这样会更符合“聚焦/散焦边界”决策的物理直觉。

##### 2) 训练稳定性：prior_beta 建议“分层/分阶段”

> 你现在的 prior 注入是 **所有 Swin stage 都用同一个 prior_beta**（通过 PatchTokenMixer 传进去）。 hybrid_fusion_net
>
> 经验上更稳的做法是：
>
> - **浅层（高分辨率）prior 强一点**：帮助边界和细粒度区域更快对齐
> - **深层（低分辨率）prior 弱一点**：避免语义先验过度支配全局结构
>

5. 问题来了：不自然的窗口组合

**窗口3包含不相邻的像素**：

- 位置13, 14（原特征图的第3行）
- 位置1, 2（原特征图的第0行）

在**原始空间**中，位置1和13不相邻，不应该互相计算注意力！这就是需要**掩码**的原因。

掩码**不是**阻止窗口间信息交换，而是**精确控制**交换的方式。它确保**只有相邻的原始窗口之间能交换信息**，而不是让所有像素随意混合。

- #### `patch_size` 的参数意义是什么？

  - **每个 token 对应原特征图上多大的一块区域（patch）**
     同时也是 **下采样倍率**（stride=patch_size）

- **将注意力头数量（num_heads）增加会有什么影响？**
  - **影响 1：表达能力变化（更“细分视角”）**
    - 多头注意力可以理解为：**同时用多种“相似度度量方式”去看同一个窗口**
    - heads 越多，模型更容易学到不同类型关系（边缘、纹理、结构、对比度等）
  - **每个 head 的维度会变小（重要）**
    - ead_dim 太小可能导致每个头的信息容量不足，反而效果不一定更好。
- **为什么 Transformer 部分对初始化很敏感？**
  - Transformer 里面有三类“敏感算子”，一旦初始权重尺度不合适，就容易训练不稳（梯度爆炸/消失、softmax 饱和等）。
